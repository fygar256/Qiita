# プログラミングの際に思ったこと

pythonのpopがちょっと使いづらい
例えば、
l=Noneまたはl=[]のとき、
```python
i=l.pop()
```
としたら、pythonはエラーを返すが、無視して、iにNoneを返して、lはそのままにしていて欲しい。
実装すると、こう
```python
>>> po=lambda l:l.pop() if l else None
>>> l=['a','b']
>>> i=po(l)
>>> l
['a']
>>> i
'b'
>>> l=[]
>>> i=po(l)
>>> l
[]
>>> i
>>> 
>>> l=None
>>> i=po(l)
>>> l
>>> i
>>>

```
NULLの処理の仕方の問題ですにゃ。
